### Contact Management Data Structure
[Source: architecture/database-schema.md#client-management-tables]
Contact model and relationships:
```typescript
interface ClientContact {
  id: number
  clientId: number
  name: string
  email?: string
  phone?: string
  role?: string
  department?: string
  isPrimary: boolean
  status: 'active' | 'inactive'
  createdAt: string
  notes?: string
}
```

### Client-Contact API Integration
[Source: docs/stories/3.1.story.md#contact-management-integration]
Contact API endpoints:
- **GET /api/clients/{id}/contacts**: List all contacts for a client
- **POST /api/clients/{id}/contacts**: Create new contact for client
- **PUT /api/clients/{id}/contacts/{contactId}**: Update contact information
- **DELETE /api/clients/{id}/contacts/{contactId}**: Delete contact
- **PUT /api/clients/{id}/contacts/{contactId}/primary**: Set as primary contact

### Contact Validation Rules
Contact validation requirements:
- **Required fields**: name, client association
- **Optional fields**: email, phone, role, department
- **Email validation**: Proper email format when provided
- **Phone validation**: Valid phone number format when provided
- **Primary contact**: Only one primary contact per client
- **Duplicate detection**: Prevent duplicate emails within client

### Contact Form Validation
Contact form validation schema:
```typescript
const contactFormSchema = z.object({
  name: z.string().min(1, 'Contact name is required').max(255),
  email: z.string().email('Invalid email format').optional().or(z.literal('')),
  phone: z.string().regex(/^[\+]?[1-9][\d]{0,15}$/, 'Invalid phone format').optional(),
  role: z.string().max(100).optional(),
  department: z.string().max(100).optional(),
  isPrimary: z.boolean().default(false),
  notes: z.string().max(500).optional(),
})
```
