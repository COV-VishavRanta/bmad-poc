## Dev Notes

### Authentication Flow Architecture
[Source: docs/stories/2.1.story.md]
- Backend provides session-based auth with HttpOnly cookies
- Frontend manages user state and role-based access control
- Session expires after 24 hours with automatic refresh capability
- Role-based permissions: HR (full access), PC (projects/clients), RM (assignments/timelines)

### Next.js Authentication Patterns
[Source: architecture/tech-stack.md#frontend-technologies]
- Use Next.js 15+ App Router with middleware for route protection
- Implement authentication context using React Context API
- Use Zustand for client-side authentication state management
- Leverage Next.js middleware for server-side route protection

### Form Validation and Management
[Source: architecture/coding-standards.md#react-best-practices]
- Use React Hook Form for form state management
- Implement Zod schemas for type-safe validation
- Add proper form accessibility (ARIA labels, focus management)
- Use Material-UI form components with consistent styling

### State Management Structure
Authentication context interface:
```typescript
interface AuthContext {
  user: User | null
  isLoading: boolean
  isAuthenticated: boolean
  login: (email: string, password: string) => Promise<void>
  logout: () => Promise<void>
  refreshSession: () => Promise<void>
  hasPermission: (permission: string) => boolean
  hasRole: (role: UserRole) => boolean
}
```

### Role-Based Access Control
[Source: docs/prd/epic-2-authentication-authorization.md]
User roles and their permissions:
- **HR**: Access to all pages and features
- **PC**: Access to projects, clients, SOWs (no user management)
- **RM**: Access to assignments, project timelines, team data
- Implement permission checking at component and route level

### Protected Route Implementation
Next.js middleware for route protection:
```typescript
// middleware.ts
export function middleware(request: NextRequest) {
  const token = request.cookies.get('clientops_session')
  const { pathname } = request.nextUrl
  
  // Check if route requires authentication
  if (protectedRoutes.includes(pathname) && !token) {
    return NextResponse.redirect(new URL('/login', request.url))
  }
  
  // Check role-based access
  const userRole = getUserRoleFromToken(token)
  if (!hasRouteAccess(pathname, userRole)) {
    return NextResponse.redirect(new URL('/unauthorized', request.url))
  }
}
```
