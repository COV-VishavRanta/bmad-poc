### SOW Form Structure and Validation
[Source: docs/stories/6.1.story.md#fte-allocation-validation-logic]
SOW form validation using Zod:
```typescript
const sowFormSchema = z.object({
  name: z.string().min(1, 'SOW name is required').max(255),
  clientId: z.number().min(1, 'Client selection is required'),
  description: z.string().max(1000).optional(),
  startDate: z.date().optional(),
  endDate: z.date().optional(),
  status: z.enum(['draft', 'active', 'completed', 'cancelled']),
  roles: z.array(z.object({
    roleName: z.enum(['Backend Developer', 'Frontend Developer', 'Designer', 'Tester', 'Business Analyst', 'Architect', 'Project Manager']),
    fteAllocation: z.number().min(0.1).max(1.0).multipleOf(0.1),
    hourlyRate: z.number().min(0).max(1000).optional(),
    description: z.string().max(500).optional()
  }))
}).refine((data) => !data.endDate || !data.startDate || data.startDate <= data.endDate, {
  message: "End date must be after start date",
  path: ["endDate"],
}).refine((data) => {
  const totalFTE = data.roles.reduce((sum, role) => sum + role.fteAllocation, 0);
  return totalFTE <= 10.0;
}, {
  message: "Total FTE allocation cannot exceed 10.0",
  path: ["roles"],
})
```

### API Integration
[Source: docs/stories/6.1.story.md#api-endpoint-specifications]
Frontend API calls for SOW management:
- **GET /api/sows**: Fetch SOWs with search and pagination
- **POST /api/sows**: Create new SOW with role allocations
- **GET /api/sows/{id}**: Fetch single SOW with roles and history
- **PUT /api/sows/{id}**: Update SOW information
- **DELETE /api/sows/{id}**: Delete SOW with dependency checking
- **GET /api/sows/{id}/roles**: Fetch role allocations
- **POST /api/sows/{id}/roles**: Add role allocation
- **PUT /api/sows/{id}/roles/{role_id}**: Update role allocation
- **GET /api/sows/{id}/history**: Fetch version history
- **GET /api/sows/{id}/versions/{version}**: Fetch specific version
