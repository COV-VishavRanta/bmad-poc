# Story 3.2: Frontend Client Management Interface

## Status
Completed

## Story
**As a** user with appropriate permissions,
**I want** a client management interface to create, view, and manage client information,
**so that** I can effectively organize and track client relationships.

## Acceptance Criteria
1. Create client listing page with search and filters
2. Implement client creation/editing forms with validation
3. Add client deactivation with confirmation dialogs
4. Implement protected routes for role-based access
5. Show client dependency warnings before deactivation
6. Display client audit history and change tracking
7. Add responsive design for desktop and tablet

### Client Management UI Design

### Role-Based Access Control
[Source: docs/prd/epic-2-authentication-authorization.md]
Client management permissions by role:
- **HR users**: Full access to all client operations (create, read, update, deactivate)
- **PC users**: Read/write access to client data and management functions
- **RM users**: Read-only access to client information for assignment context
- Implement permission checking at component and route levels

### Form Validation Schema
Client form validation using Zod:
```typescript
const clientFormSchema = z.object({
  name: z.string().min(1, 'Client name is required').max(255),
  status: z.enum(['active', 'inactive']),
  relationType: z.enum(['Customer', 'Partner', 'Internal']),
  projectManagementTool: z.string().optional(),
  comments: z.string().max(1000).optional(),
})
```

### Client Dependency Checking
[Source: docs/stories/3.1.story.md#business-logic-rules]
Before client deactivation, check for:
- Active projects associated with the client
- Ongoing assignments linked to client projects
- Active SOWs belonging to the client
- Display dependency summary in confirmation dialog

### Client Status Management
Client status indicators and management:
- **Active**: Green badge, full functionality available
- **Inactive**: Red badge, limited operations, view-only mode
- **Status toggle**: Confirmation dialog with dependency check
- **Visual feedback**: Clear status indicators throughout UI

### API Integration
[Source: docs/stories/3.1.story.md#api-endpoint-specifications]
Frontend API calls for client management:
- **GET /api/clients**: Fetch clients with search and pagination
- **POST /api/clients**: Create new client
- **PUT /api/clients/{id}**: Update client information
- **DELETE /api/clients/{id}**: Deactivate client
- **GET /api/clients/{id}/history**: Fetch client audit history

### State Management
Client state management with Zustand:
```typescript
interface ClientStore {

  updateClient: (id: number, data: UpdateClientData) => Promise<void>
  deactivateClient: (id: number) => Promise<void>
  setSearchParams: (params: ClientSearchParams) => void
}
```
