### Project-Group Association Implementation
[Source: architecture/database-schema.md#groups]
Group-project relationship requirements:
- **Group Model**: groups table with client_id, sow_id, start_date, end_date
- **Project Association**: projects.group_id foreign key (optional)
- **Date Validation**: Project dates must fall within group date range
- **Business Rules**: Group names unique within client scope
- **Capacity Management**: Group FTE allocation summary across projects

### Analytics and Reporting Data Models
[Source: architecture/database-schema.md#project-management-tables]
Project analytics data structure:
```sql
-- Project metrics and KPIs
CREATE TABLE project_metrics (
    id INT PRIMARY KEY AUTO_INCREMENT,
    project_id INT NOT NULL,
    metric_type ENUM('duration', 'budget', 'team_size', 'completion_rate') NOT NULL,
    metric_value DECIMAL(10,2) NOT NULL,
    measurement_date DATE NOT NULL,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- Project milestones
CREATE TABLE project_milestones (
    id INT PRIMARY KEY AUTO_INCREMENT,
    project_id INT NOT NULL,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    due_date DATE NOT NULL,
    status ENUM('pending', 'in_progress', 'completed', 'overdue') DEFAULT 'pending',
    completion_date DATE NULL,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);
```
