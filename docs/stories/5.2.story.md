**I want** a group management interface to organize and manage project groups,
**so that** I can effectively structure related projects and maintain organizational hierarchy.

### Group Form Validation Schema
Group form validation using Zod:
```typescript
const groupFormSchema = z.object({
  name: z.string().min(1, 'Group name is required').max(255),
  clientId: z.number().min(1, 'Client selection is required'),
  sowId: z.number().optional(),
  startDate: z.date(),
  endDate: z.date(),
  description: z.string().max(1000).optional(),
}).refine((data) => data.startDate <= data.endDate, {
  message: "End date must be after start date",
  path: ["endDate"],
})
```

### Role-Based Access Control Implementation
[Source: docs/prd/epic-2-authentication-authorization.md]
Group management permissions:
- **HR users**: Full access to all group operations and cross-client visibility
- **PC users**: Full read/write access to group management and creation
- **RM users**: Read-only access to group information for assignment context
- **Component-level**: Hide/show buttons, forms, and sections based on permissions
- **Data filtering**: Filter group data based on user role and client access

### API Integration
[Source: docs/stories/5.1.story.md#api-endpoint-specifications]
Frontend API calls for group management:
- **GET /api/groups**: Fetch groups with search and pagination
- **POST /api/groups**: Create new group with validation
- **GET /api/groups/{id}**: Fetch single group with projects and analytics
- **PUT /api/groups/{id}**: Update group information
- **DELETE /api/groups/{id}**: Delete group with dependency checking
- **POST /api/groups/{id}/projects**: Add project to group
- **DELETE /api/groups/{id}/projects/{project_id}**: Remove project from group
