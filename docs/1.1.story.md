### Data Models

[Source: architecture/data-models.md#user]

- User model with id (UUID), email, name, image, createdAt, updatedAt
- Relationships: One-to-many with Posts, Comments, Claps

[Source: architecture/data-models.md#post]

- Post model with id (UUID), title, content, excerpt, slug, authorId, published, readingTime, timestamps
- Relationships: Many-to-one with User, One-to-many with Comments and Claps

[Source: architecture/data-models.md#comment]

- Comment model with threading support via parentId for nested discussions
- Relationships: Many-to-one with User and Post, self-referential for threading

[Source: architecture/data-models.md#clap]

- Clap model for Medium-style appreciation with count limits (max 50 per user per post)
- Unique constraint on (userId, postId) to prevent duplicate records

### API Specifications

[Source: architecture/backend-architecture.md#authentication-configuration]

- NextAuth.js v5+ integration with Google OAuth provider
- JWT session strategy with 30-day expiration
- Prisma adapter for database session storage

[Source: architecture/backend-architecture.md#trpc-authentication-middleware]

- tRPC v10+ for end-to-end type safety
- Authentication middleware with protected procedures
- Superjson transformer for enhanced data serialization

### Component Specifications

[Source: architecture/frontend-architecture.md#component-organization]

- App Router structure with route groups for authentication and dashboard
- Component organization: ui/ (shadcn/ui), layout/, posts/, comments/, engagement/, auth/
- State management using Zustand for draft management and UI state

[Source: architecture/frontend-architecture.md#protected-route-pattern]

- Middleware-based route protection using NextAuth.js
- Client-side AuthGuard component for additional protection
- Route groups for authentication and dashboard layouts

### File Locations

[Source: architecture/frontend-architecture.md#component-organization]

- `src/app/` - Next.js App Router pages and layouts
- `src/components/` - Reusable UI components organized by domain
- `src/lib/` - Utility libraries including auth.ts, trpc.ts, utils.ts
- `src/hooks/` - Custom React hooks for data management
- `src/stores/` - Zustand state management stores
- `src/types/` - TypeScript type definitions

[Source: architecture/backend-architecture.md#service-organization]

- `src/server/api/` - tRPC routers and configuration
- `src/server/db/` - Database schema and migrations
- `src/server/services/` - Business logic services
- `src/server/middleware/` - Request processing middleware

### Testing Requirements

[Source: architecture/tech-stack.md#technology-stack-table]

- Vitest + Testing Library for frontend unit and integration testing
- Vitest for backend API route and business logic testing
- Playwright for end-to-end testing of critical user journeys
- Testing configuration must support TypeScript and Next.js App Router

### Technical Constraints

[Source: architecture/tech-stack.md#technology-stack-table]

- Next.js 15+ with App Router (mandatory)
- TypeScript 5.0+ for end-to-end type safety
- PostgreSQL 15+ for database (Supabase hosted)
- Prisma ORM for database operations
- Tailwind CSS 3.4+ for styling
- tRPC 10.0+ for API layer

[Source: architecture/backend-architecture.md#connection-management]

- Singleton Prisma client pattern for serverless optimization
- Connection pooling configuration for Supabase PostgreSQL
- Environment variable validation for database connections
